# End-to-End DevOps Pipeline: 3-Tier Web Application 🚀

This project demonstrates a complete, automated End-to-End DevOps pipeline for a 3-tier application (Frontend, Backend, Database). It covers everything from Infrastructure as Code (IaC) and Configuration Management to CI/CD, Container Orchestration, and Monitoring.

## 🛠️ Technology Stack

* **Version Control:** Git & GitHub
* **Containerization:** Docker & Docker Compose
* **Infrastructure as Code (IaC):** Terraform
* **Cloud Provider:** AWS (EC2, VPC, Security Groups)
* **Configuration Management:** Ansible
* **CI/CD Pipeline:** Jenkins
* **Container Orchestration:** Kubernetes (K3s)
* **Monitoring & Observability:** Prometheus & Grafana
* **Application:** Node.js (Backend), HTML/JS (Frontend), MongoDB (Database)

## 📁 Project Structure

```text
devops-project/
├── frontend/           # Frontend source code and Dockerfile
├── backend/            # Backend source code and Dockerfile
├── terraform/          # IaC scripts to provision AWS EC2 instances
├── ansible/            # Playbooks for server config, K8s deploy, and Monitoring
├── k8s/                # Kubernetes Manifests (Deployments & Services)
├── Jenkinsfile         # Declarative CI/CD Pipeline script
└── docker-compose.yml  # Local testing environment
```

## 📋 Prerequisites
Before running this project, ensure you have the following installed and configured on your local machine:

AWS Account and AWS CLI configured (aws configure).

Terraform installed.

Ansible installed.

Docker Hub Account (for storing container images).

An AWS SSH Key Pair named devops-key.pem stored in ~/.ssh/.

## 🚀 Step-by-Step Execution Guide
Step 1: Clone the Repository
Bash
git clone [https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git)
cd devops-project
Step 2: Provision Infrastructure (Terraform)
Navigate to the terraform directory to create the Jenkins and Kubernetes servers on AWS.

```Bash
cd terraform
terraform init
terraform plan
terraform apply --auto-approve
```
Note down the output Public IPs for the Jenkins and Kubernetes servers.

Step 3: Configure Servers (Ansible)
Update the ansible/inventory.ini file with the IPs generated by Terraform. Then, run the setup playbook to install Docker and Jenkins.

```Bash
cd ../ansible
ansible-playbook -i inventory.ini setup.yml
```

Step 4: Setup CI/CD Pipeline (Jenkins)
Access Jenkins at http://<JENKINS_SERVER_IP>:8080.


Retrieve the initial admin password via SSH.

Install the Docker Pipeline plugin.

Add your Docker Hub credentials (ID: dockerhub-creds).

Create a new Pipeline job, link it to this GitHub repository, and point it to the Jenkinsfile.

Run the build to create and push Docker images.

Step 5: Deploy to Kubernetes
Update the image names in k8s/frontend.yaml and k8s/backend.yaml with your Docker Hub username. Then, use Ansible to install K3s and deploy the application.

```Bash
ansible-playbook -i inventory.ini deploy_k8s.yml
```

Step 6: Setup Monitoring (Prometheus & Grafana)
Deploy the kube-prometheus-stack using Helm via Ansible.

```Bash
ansible-playbook -i inventory.ini deploy_monitoring.yml
```

🌐 Accessing the Services
Once everything is deployed successfully, you can access the services using the Kubernetes Server IP:

🌐 3-Tier Application: http://<K8S_SERVER_IP>:30080

📊 Grafana Dashboard: http://<K8S_SERVER_IP>:30300 (Default Login: admin / PASSWORD IN OUTPUT OF ANSIBLE)

⚙️ Jenkins CI/CD: http://<JENKINS_SERVER_IP>:8080

🧹 Clean Up (Important)
To avoid unexpected AWS charges, destroy the infrastructure once you are done testing:

```Bash
cd terraform
terraform destroy --auto-approve
```
